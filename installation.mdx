Elasticsearch ships as a Docker image, a tar.gz/zip archive, and native packages for Linux (Debian and RPM). All distributions bundle a compatible JDK — you do not need to install Java separately.

<Note>
  Elasticsearch bundles OpenJDK in every distribution. The bundled JDK is the recommended runtime. If you supply your own JDK via `JAVA_HOME`, make sure it meets the version requirements listed in the [supported platforms matrix](https://www.elastic.co/support/matrix#matrix_jvm).
</Note>

<Tabs>

<Tab title="Docker">

## Docker

Docker is the fastest way to run a single-node Elasticsearch instance for local development or testing.

<Warning>
  Setting `discovery.type=single-node` disables the production-mode bootstrap checks and is **not suitable for production**. A production cluster must use a multi-node configuration with proper discovery settings.
</Warning>

<Steps>

<Step title="Pull the image">

Replace `8.x` with the target version (e.g., `8.17.0`):

```bash
docker pull docker.elastic.co/elasticsearch/elasticsearch:8.x
```

</Step>

<Step title="Start a single-node container">

```bash
docker run --name elasticsearch \
  -p 9200:9200 \
  -e "discovery.type=single-node" \
  -e "ELASTIC_PASSWORD=changeme" \
  -e "xpack.security.enabled=true" \
  docker.elastic.co/elasticsearch/elasticsearch:8.x
```

| Environment variable | Description |
|---|---|
| `discovery.type=single-node` | Skips multi-node discovery; required for a standalone container. |
| `ELASTIC_PASSWORD` | Password for the built-in `elastic` superuser. Must be at least 6 characters. |
| `xpack.security.enabled` | Enables authentication and TLS. Set to `false` to disable (not recommended). |

</Step>

<Step title="Verify the node is running">

```bash
curl -u elastic:changeme http://localhost:9200
```

A successful response contains `"tagline": "You Know, for Search"`.

</Step>

</Steps>

### Multi-node cluster with Docker Compose

For a three-node cluster with TLS enabled, use the example `docker-compose.yml` from the Elasticsearch repository. Copy both files to a working directory, fill in passwords in `.env`, then run:

```bash
docker compose up -d
```

The Compose file starts three Elasticsearch nodes (`es01`, `es02`, `es03`) and a Kibana instance. Each node has:

- TLS certificates generated by `elasticsearch-certutil`
- `bootstrap.memory_lock=true` to prevent heap from swapping
- `xpack.security.enabled=true` with per-node certificate keypairs

## Linux

<Tabs>

<Tab title="tar.gz">

### Install from archive (tar.gz)

<Steps>

<Step title="Download and extract">

```bash
curl -O https://artifacts.elastic.co/downloads/elasticsearch/elasticsearch-8.x.x-linux-x86_64.tar.gz
tar -xzf elasticsearch-8.x.x-linux-x86_64.tar.gz
cd elasticsearch-8.x.x/
```

</Step>

<Step title="Configure (optional)">

The main configuration file is `config/elasticsearch.yml`. For a basic single-node setup, no changes are required. Common settings:

```yaml
cluster.name: my-cluster
node.name: node-1
network.host: 0.0.0.0
http.port: 9200
discovery.type: single-node
```

</Step>

<Step title="Start Elasticsearch">

```bash
./bin/elasticsearch
```

Run as a background daemon with:

```bash
./bin/elasticsearch -d -p pid
```

</Step>

<Step title="Verify">

```bash
curl -u elastic:<password> http://localhost:9200
```

The initial password for the `elastic` user is printed to the terminal on first start. To reset it, run:

```bash
./bin/elasticsearch-reset-password -u elastic
```

</Step>

</Steps>

</Tab>

<Tab title="apt (Debian/Ubuntu)">

### Install with apt

<Steps>

<Step title="Import the Elastic GPG key">

```bash
wget -qO - https://artifacts.elastic.co/GPG-KEY-elasticsearch | \
  sudo gpg --dearmor -o /usr/share/keyrings/elasticsearch-keyring.gpg
```

</Step>

<Step title="Add the apt repository">

```bash
sudo apt-get install apt-transport-https
echo "deb [signed-by=/usr/share/keyrings/elasticsearch-keyring.gpg] https://artifacts.elastic.co/packages/8.x/apt stable main" | \
  sudo tee /etc/apt/sources.list.d/elastic-8.x.list
```

</Step>

<Step title="Install Elasticsearch">

```bash
sudo apt-get update && sudo apt-get install elasticsearch
```

</Step>

<Step title="Configure and start">

```bash
sudo systemctl daemon-reload
sudo systemctl enable elasticsearch
sudo systemctl start elasticsearch
```

</Step>

<Step title="Verify">

```bash
sudo systemctl status elasticsearch
curl -u elastic:<password> http://localhost:9200
```

</Step>

</Steps>

</Tab>

<Tab title="yum/dnf (RHEL/CentOS)">

### Install with yum or dnf

<Steps>

<Step title="Import the Elastic GPG key">

```bash
sudo rpm --import https://artifacts.elastic.co/GPG-KEY-elasticsearch
```

</Step>

<Step title="Add the yum repository">

Create `/etc/yum.repos.d/elasticsearch.repo`:

```ini
[elasticsearch]
name=Elasticsearch repository for 8.x packages
baseurl=https://artifacts.elastic.co/packages/8.x/yum
gpgcheck=1
gpgkey=https://artifacts.elastic.co/GPG-KEY-elasticsearch
enabled=1
autorefresh=1
type=rpm-md
```

</Step>

<Step title="Install Elasticsearch">

```bash
sudo yum install --enablerepo=elasticsearch elasticsearch
# or with dnf:
sudo dnf install elasticsearch
```

</Step>

<Step title="Configure and start">

```bash
sudo systemctl daemon-reload
sudo systemctl enable elasticsearch
sudo systemctl start elasticsearch
```

</Step>

<Step title="Verify">

```bash
sudo systemctl status elasticsearch
curl -u elastic:<password> http://localhost:9200
```

</Step>

</Steps>

## macOS

<Tab title="Homebrew">

### Install with Homebrew

<Steps>

<Step title="Tap the Elastic Homebrew formulae">

```bash
brew tap elastic/tap
```

</Step>

<Step title="Install Elasticsearch">

```bash
brew install elastic/tap/elasticsearch-full
```

This installs the full distribution including the bundled JDK.

</Step>

<Step title="Start Elasticsearch">

```bash
elasticsearch
```

Or start it as a background service:

```bash
brew services start elastic/tap/elasticsearch-full
```

</Step>

<Step title="Verify">

```bash
curl -u elastic:<password> http://localhost:9200
```

The initial password is printed on first start. Reset it with:

```bash
elasticsearch-reset-password -u elastic
```

</Step>

</Steps>

<Tab title="Windows">

## Windows

<Steps>

<Step title="Download the zip archive">

Download the Windows zip from the [Elasticsearch downloads page](https://www.elastic.co/downloads/elasticsearch) and extract it:

```powershell
Expand-Archive -Path elasticsearch-8.x.x-windows-x86_64.zip -DestinationPath C:\elasticsearch
```

</Step>

<Step title="Configure (optional)">

Open `C:\elasticsearch\elasticsearch-8.x.x\config\elasticsearch.yml` and adjust settings as needed. For a local single-node setup, the defaults are sufficient.

</Step>

<Step title="Start Elasticsearch">

From PowerShell:

```powershell
C:\elasticsearch\elasticsearch-8.x.x\bin\elasticsearch.bat
```

</Step>

<Step title="Verify">

```powershell
curl.exe -u elastic:<password> http://localhost:9200
```

The initial password for the `elastic` user is printed to the terminal on first start. Reset it with:

```powershell
C:\elasticsearch\elasticsearch-8.x.x\bin\elasticsearch-reset-password.bat -u elastic
```

</Step>

</Steps>

<Note>
  Running Elasticsearch as a Windows service is supported via the `elasticsearch-service.bat` helper. See the [Elasticsearch service documentation](https://www.elastic.co/guide/en/elasticsearch/reference/current/zip-windows.html) for details.
</Note>
